# GoBlueWine

> Читать эту страницу на английском: [English version](README.md)

GoBlueWine строит модульную экосистему вокруг Go для создания полноценных продуктов, а не только отдельных пакетов.

Цель простая: сделать Go удобным для разработки реальных систем end to end, с переиспользуемыми модулями для структуры приложений, model-driven development, сайтов, сервисов, tooling и более высокоуровневых developer workflows.

## Что мы создаём

- `BlueWine` это основной framework-репозиторий для разработки приложений на Go.
- `RPLang` это язык описания моделей и toolchain для генерации типизированных систем вокруг общих моделей.
- Вся экосистема в целом направлена на переиспользуемые модули для API, сайтов, внутренних инструментов и полноценных приложений.
- Мы также исследуем более высокоуровневые идеи вроде iGo и другие product-oriented модули, которые делают разработку на Go более цельной и связной.

## Что для нас важно

- Ясная архитектура вместо случайной сложности.
- Сильная типизация и generated glue там, где это реально экономит работу.
- Модульный дизайн, чтобы маленькие инструменты могли вырастать в большие системы.
- Практичный developer experience для команд, которые строят реальные продукты на Go.
- Экосистема, в которой отдельные инструменты всё равно чисто стыкуются друг с другом.

## Зачем

Go отлично подходит по производительности, деплою и сопровождаемости, но многим командам всё ещё приходится собирать свой стек вручную.

GoBlueWine существует, чтобы сделать эту историю лучше: связный набор инструментов и модулей для описания моделей, генерации кода, структуры приложений и разработки продуктов с меньшим количеством boilerplate и fragmentation.

## Текущие репозитории

- [BlueWine](https://github.com/GoBlueWine/BlueWine)
  Основной framework-репозиторий для разработки приложений.

- [RPLang](https://github.com/GoBlueWine/RPLang)
  Язык и toolchain для описания моделей и генерации полезного кода вокруг них.

## Видение

Мы хотим, чтобы GoBlueWine вырос в практичную development platform для Go:

- один раз описываешь модели
- генерируешь вокруг них повторяющиеся слои
- компонуешь модули вместо переписывания одной и той же инфраструктуры
- строишь сайты, сервисы, инструменты и приложения в одной связанной экосистеме

## Следить за развитием

Организация пока ещё на раннем этапе, но направление уже хорошо видно: построить серьёзную модульную основу для product development на Go.

Если тебе интересен проект, начни с репозиториев выше и наблюдай за организацией: по мере роста экосистемы будут появляться новые модули.
