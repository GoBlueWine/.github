# GoBlueWine

GoBlueWine is building a modular ecosystem around Go for writing complete products, not just isolated packages.

The goal is simple: make Go comfortable for building real systems end to end, with reusable modules for application structure, model-driven development, websites, services, tooling, and higher-level developer workflows.

## What We Build

- `BlueWine` is the main framework repository for application development in Go.
- `RPLang` is a model description language and code generation toolchain for building typed systems around shared models.
- The broader ecosystem is aimed at reusable modules for APIs, websites, internal tools, and full applications.
- We are also exploring higher-level ideas such as iGo and other product-oriented modules that make Go development feel more complete and integrated.

## What We Care About

- Clear architecture instead of accidental complexity.
- Strong typing and generated glue where it saves real work.
- Modular design, so small tools can grow into larger systems.
- Practical developer experience for teams building real products in Go.
- An ecosystem where separate tools still fit together cleanly.

## Why

Go is excellent for performance, deployment, and maintainability, but many teams still end up assembling their whole stack by hand.

GoBlueWine exists to make that story better: a coherent set of tools and modules for describing models, generating code, structuring applications, and building products with less boilerplate and less fragmentation.

## Current Repositories

- [BlueWine](https://github.com/GoBlueWine/BlueWine)
  The main framework repository for application development.

- [RPLang](https://github.com/GoBlueWine/RPLang)
  A language and toolchain for describing models and generating useful code around them.

## Vision

We want GoBlueWine to grow into a practical development platform for Go:

- define your models once
- generate the repetitive layers around them
- compose modules instead of rewriting the same infrastructure
- build websites, services, tools, and applications in one connected ecosystem

## Follow the Work

The organization is still early, but the direction is already clear: build a serious, modular foundation for product development in Go.

If you are interested in the project, start with the repositories above and watch the organization for new modules as the ecosystem grows.
